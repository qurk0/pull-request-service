Пока что список принятых решений:
1. Переменная LOGLEVEL в ENV-переменных, по умолчанию будет стоять LevelInfo, если вдруг захочется взять другой уровень логирования - в docker-compose.yml меняем LOGLEVEL на уровень: 'warn', 'error', 'debug' (пока что только строчные литеры работают, в дальнейшем может будет свободный регистр в переменных окружения и буду всё к lowerCase приводить, пока не до этого)

2. В документацию добавил ошибку UNAUTHORIZED для ошибки 401, 

3. Добавил возврат ошибки 401 для эндпоинто, где указано поле security (без токена доступа по документации будет возвращаться ошибка). Принял это решение, так как везде, где требуется токен, в случае его отсутствия или невалидность мы должны возвращать ошибку.

4. В эндпоинте /team/add заменил 400 на 409 - команда уже существует, значит она найдена, но происходит конфликт с состоянием сервера, по документации статус-кодов:

"409 Conflict

    Запрос конфликтует с текущим состоянием сервера."

Поэтому решил заменить.
